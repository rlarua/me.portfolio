## 📐 1. Design Principles

### **Architectural Clarity** (구조적 명확성)

불필요한 장식을 배제하고 **그리드 기반의 견고한 구조**를 지향합니다.  
정보의 위계가 명확하게 드러나도록 설계합니다.

### **Minimalist & Premium** (미니멀 & 프리미엄)

넓은 여백(Whitespace)과 높은 대비를 통해 **콘텐츠에 집중**할 수 있는 고급스러운 룩앤필을 유지합니다.

### **Result-Oriented** (결과 중심)

특정 강조색(Sunset Gold)을 통해 **성과와 수치를 직관적으로 노출**합니다.  
사용자가 가장 먼저 봐야 할 정보에 시선을 유도합니다.

#### **Color Usage Rules**

```
✅ DO:
- Sunset Gold: 성과 수치, 프로젝트 결과, CTA 버튼
- Tech Cyan: 기술 태그, 인터랙티브 요소
- Charcoal Black: 제목, 본문 텍스트 (가독성 최우선)

❌ DON'T:
- 3가지 이상의 accent color 혼용
- 낮은 대비의 텍스트 배치 (접근성 저해)
- Sunset Gold를 배경색으로 과도하게 사용
```

## ✍️ 3. Typography

### **Font Family**

| 언어 | Primary | Fallback |
|:---|:---|:---|
| **한글** | Pretendard | Apple SD Gothic Neo, Malgun Gothic |
| **영문/숫자** | Inter | -apple-system, system-ui |
| **코드** | Fira Code | Consolas, Monaco |

### **Font Weights & Sizes**

```css
/* Headers */
H1: 48px / ExtraBold (800) / Line-height: 1.2
H2: 36px / Bold (700) / Line-height: 1.3
H3: 24px / SemiBold (600) / Line-height: 1.4

/* Body */
Body Large: 18px / Regular (400) / Line-height: 1.7
Body Default: 16px / Regular (400) / Line-height: 1.6
Caption: 14px / Medium (500) / Line-height: 1.5
```

### **Typography Hierarchy**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  H1: Section Title (섹션 타이틀)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    H2: Sub-section (서브 섹션)
    ────────────────────────────────────
      H3: Component Title (컴포넌트 제목)
      
      Body: 설명 텍스트는 충분한 line-height(1.6~1.7)로
      가독성을 확보하며, 단락 간 간격은 1.5em 이상 유지
      
      Caption: 추가 정보, 메타데이터
```

## 🧩 4. UI Components

### **4.1. Cards**

```
┌─────────────────────────────────────┐
│                                     │
│  [Icon/Image]                       │
│                                     │
│  Card Title (H3, Charcoal Black)    │
│  ─────────────────────────────      │
│  설명 텍스트 (Body, Slate Gray)      │
│                                     │
│  [CTA Button or Link]               │
│                                     │
└─────────────────────────────────────┘

스타일:
- Background: #FFFFFF
- Border: 1px solid #E5E7EB 또는 subtle shadow
- Border Radius: 8px~12px
- Padding: 24px~32px
- Hover: 미세한 scale(1.02) + shadow 강화
```

### **4.2. Buttons**

#### **Primary Button** (주요 액션)

```css
Background: #FF9F0A (Sunset Gold)
Text: #FFFFFF (White)
Padding: 12px 24px
Border Radius: 6px
Font Weight: 600

Hover: Background #FF8C00 (약간 어둡게)
```

#### **Secondary Button** (보조 액션)

```css
Background: transparent
Border: 2px solid #1A1A1A (Charcoal Black)
Text: #1A1A1A
Padding: 12px 24px
Border Radius: 6px

Hover: Background #1A1A1A, Text #FFFFFF
```

#### **Text Link** (텍스트 링크)

```css
Text: #00E5FF (Tech Cyan)
Text Decoration: none
Font Weight: 500

Hover: Underline + 약간 밝은 Cyan
```

### **4.3. Input Fields**

```css
Border: 1px solid #D1D5DB
Border Radius: 6px
Padding: 10px 16px
Font Size: 16px

Focus:
  Border: 2px solid #00E5FF
  Shadow: 0 0 0 3px rgba(0, 229, 255, 0.1)
```

### **4.4. Tags / Badges**

```css
Background: #F3F4F6 (기본 회색)
Text: #374151
Padding: 4px 12px
Border Radius: 12px (Pill shape)
Font Size: 14px

기술 스택 태그:
  Background: rgba(0, 229, 255, 0.1)
  Text: #00E5FF
  Border: 1px solid #00E5FF
```

## 📏 5. Layout & Spacing

### **Grid System**

```
Desktop (>1024px): 12 Column Grid
Tablet (768px~1023px): 8 Column Grid  
Mobile (<767px): 4 Column Grid

Max Content Width: 1200px (중앙 정렬)
Side Margin: 24px (mobile), 48px (tablet), 80px (desktop)
```

### **Spacing Scale** (8px 기반)

| Name | Value | Usage |
|:---|:---|:---|
| **xs** | 4px | 작은 간격 (아이콘-텍스트) |
| **sm** | 8px | 요소 간 최소 간격 |
| **md** | 16px | 기본 여백 (버튼, 카드 내부) |
| **lg** | 24px | 섹션 내부 간격 |
| **xl** | 32px | 컴포넌트 간 간격 |
| **2xl** | 48px | 큰 섹션 간격 |
| **3xl** | 64px | 메인 섹션 구분 |

### **Whitespace Philosophy**

```
원칙: "정보가 숨쉴 수 있는 공간"

✅ DO:
- 섹션 간 최소 48px 이상의 여백
- 카드 내부 충분한 padding (24px~32px)
- 텍스트 블록 간 1.5em 간격

❌ DON'T:
- 모든 공간을 콘텐츠로 채우려 시도
- 밀집된 정보 배치 (피로도 증가)
```

## 🎬 6. Motion & Animation

### **Animation Principles**

```css
/* 부드러운 전환 */
transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);

/* 카드 호버 */
transform: translateY(-4px);
box-shadow: 0 10px 30px rgba(0, 0, 0, 0.1);

/* 페이드인 */
opacity: 0 → 1 (duration: 0.5s)
```

### **Performance Rules**

```
✅ 애니메이션 허용: transform, opacity
❌ 성능 저하: width, height, margin, padding
```

## ♿ 7. Accessibility

### **Color Contrast**

```
WCAG 2.1 Level AA 준수
- 일반 텍스트: 최소 4.5:1
- 큰 텍스트(18px+): 최소 3:1

검증 도구: https://webaim.org/resources/contrastchecker/
```

### **Keyboard Navigation**

```
모든 인터랙티브 요소는 Tab 키로 접근 가능
Focus 상태를 명확히 표시 (outline 또는 shadow)
```

## 📱 8. Responsive Breakpoints

```css
/* Mobile First Approach */

Mobile:     < 768px   (기본)
Tablet:     768px ~ 1023px
Desktop:    1024px ~ 1439px
Large:      ≥ 1440px

/* 미디어 쿼리 예시 */
@media (min-width: 768px) { /* Tablet */ }
@media (min-width: 1024px) { /* Desktop */ }
@media (min-width: 1440px) { /* Large */ }
```

## 🛠️ 9. Implementation Checklist

디자인 가이드 적용 시 확인 사항:

- [ ] Primary/Accent 컬러만 사용 (추가 색상 X)
- [ ] Pretendard/Inter 폰트 로드 확인
- [ ] 모든 텍스트 대비 4.5:1 이상
- [ ] 카드 Border Radius: 8~12px
- [ ] 버튼 호버 효과 구현
- [ ] 섹션 간 최소 48px 여백
- [ ] 12 Column Grid 구조
- [ ] Mobile/Tablet/Desktop 반응형 테스트
- [ ] Keyboard 네비게이션 가능
- [ ] 애니메이션 0.3s 이하 (성능)
